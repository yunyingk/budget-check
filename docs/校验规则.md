# 预算校验规则

## 单据上的关键字段

每个报销单据有 4 个字段：
- **费用性质**：业务 / 管理 / 生产 / 未知
- **成本中心**：一个ID
- **费用档案**：一个ID
- **项目**：一个ID（可空）

## 校验流程

### 第一步：看费用性质

根据单据上的"费用性质"字段，走不同分支：

---

#### 分支A：费用性质 = 业务 或 管理

只校验两项：**成本中心** + **费用档案**

| 成本中心 | 费用档案 | 结果 |
|---------|---------|------|
| 在预算内 | 在预算内 | 通过 |
| 不在预算内 | 在预算内 | 拒绝：成本中心不在预算内 |
| 在预算内 | 不在预算内 | 拒绝：费用档案不在预算内 |
| 不在预算内 | 不在预算内 | 拒绝：成本中心不在预算内、费用档案不在预算内 |

> 业务和管理费用 **不需要看项目**。

---

#### 分支B：费用性质 = 生产

先看项目字段，再分两路：

**B1：项目名叫"公摊成本"（豁免）**

| 成本中心 | 费用档案 | 结果 |
|---------|---------|------|
| 在预算内 | 在预算内 | 通过（公摊豁免） |
| 缺 | 缺 | 拒绝：列出缺少的项 |

> 项目是"公摊成本"时，项目本身不校验，只看成本中心和费用档案。

**B2：项目是普通项目**

| 成本中心 | 费用档案 | 项目 | 结果 |
|---------|---------|------|------|
| 在预算内 | 在预算内 | 在预算内 | 通过 |
| 在预算内 | 在预算内 | 不在预算内 | 拒绝：项目不在预算内 |
| 缺 | 缺 | 缺 | 拒绝：列出所有缺少的项 |

> 普通生产费用三项都得过才算通过。

---

#### 分支C：费用性质 = 未知

直接拒绝，提示"未配置的性质ID"。

---

### 第二步："在预算内"怎么判定

成本中心、费用档案、项目，判定逻辑相同：

1. 用这个ID去内存缓存里查（不限维度类型）
2. 找到 → 通过
3. 没找到 → 调合思 API 查它的上级，再查上级的上级……最多追 5 层
4. 追到底还没找到 → 不在预算内

> **项目有个特例**：如果缓存里没找到，但项目名含"公摊成本"，视为通过。
