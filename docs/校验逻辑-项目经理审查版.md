# 预算校验逻辑说明（项目经理审查版）

## 一、业务流程概述

```
员工提交报销单
      ↓
合思审批流到达预算校验节点
      ↓
本服务接收校验请求
      ↓
判断单据是否在预算范围内
      ↓
┌─────────┴─────────┐
│                   │
通过              拒绝
│                   │
↓                   ↓
审批继续          审批终止，退回给员工
```

## 二、单据关键字段

每个报销单据上有 4 个关键字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **费用性质** | 决定走哪个校验分支 | 业务 / 管理 / 生产 |
| **成本中心** | 员工所属的成本中心 | "北京研发中心" |
| **费用档案** | 费用的具体类型 | "差旅费-住宿" |
| **项目** | 关联的项目（生产费用必填） | "XX产品研发" |

## 三、校验逻辑分支

### 分支一：业务/管理费用

**适用范围**：费用性质为"业务"或"管理"的单据

**校验规则**：只看 **成本中心** + **费用档案**，不看项目

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   成本中心在预算包内？                               │
│        │                                            │
│   ┌────┴────┐                                       │
│   是       否 → 拒绝：成本中心不在预算内            │
│   │                                                    │
│   ↓                                                    │
│   费用档案在预算包内？                               │
│        │                                            │
│   ┌────┴────┐                                       │
│   是       否 → 拒绝：费用档案不在预算内            │
│   │                                                    │
│   ↓                                                    │
│   通过                                               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**示例**：
- 北京研发中心 + 差旅费住宿 → ✅ 通过
- 北京研发中心 + 招待费 → ❌ 拒绝（招待费不在预算内）
- 上海销售部 + 差旅费住宿 → ❌ 拒绝（上海销售部不在预算内）

---

### 分支二：生产费用（普通项目）

**适用范围**：费用性质为"生产"，且项目不是"公摊成本"

**校验规则**：需要 **项目** + **成本中心** + **费用档案** 三项都通过

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   项目字段是否为空？                                 │
│        │                                            │
│   ┌────┴────┐                                       │
│   是       否                                       │
│   │         │                                       │
│   ↓         ↓                                       │
│  拒绝    项目在预算包内？                           │
│           │                                         │
│      ┌────┴────┐                                    │
│      是       否 → 拒绝：项目不在预算内             │
│      │                                              │
│      ↓                                              │
│   项目下有成本中心要求？                            │
│           │                                         │
│      ┌────┴────┐                                    │
│      否       是                                    │
│      │         │                                    │
│      ↓         ↓                                    │
│   (跳过)    成本中心在项目预算包内？                │
│                  │                                  │
│             ┌────┴────┐                             │
│             是       否 → 拒绝：成本中心不在项目预算包内 │
│             │                                       │
│             ↓                                       │
│          继续校验成本中心 + 费用档案（同分支一）     │
│                  │                                  │
│             ┌────┴────┐                             │
│             通过     拒绝                           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**示例**：
- XX项目 + 北京研发中心 + 差旅费住宿 → ✅ 通过
- XX项目 + 上海销售部 + 差旅费住宿 → ❌ 拒绝（上海销售部不在XX项目下）
- XX项目 + 北京研发中心 + 招待费 → ❌ 拒绝（招待费不在预算内）

---

### 分支三：生产费用（公摊成本 - 豁免）

**适用范围**：费用性质为"生产"，且项目是"公摊成本"

**校验规则**：**项目不校验**，只看成本中心 + 费用档案（同业务/管理费用）

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   项目 = 公摊成本（或项目ID在豁免名单中）           │
│        │                                            │
│        ↓                                            │
│   跳过项目校验                                      │
│        │                                            │
│        ↓                                            │
│   按"业务/管理"规则校验成本中心 + 费用档案          │
│        │                                            │
│   ┌────┴────┐                                       │
│   通过     拒绝                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**示例**：
- 公摊成本 + 北京研发中心 + 差旅费住宿 → ✅ 通过（项目不校验）
- 公摊成本 + 北京研发中心 + 招待费 → ❌ 拒绝（招待费不在预算内）

---

### 分支四：费用性质未知

**适用范围**：费用性质不在已配置的三种（业务/管理/生产）中

**校验规则**：**直接拒绝**

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   费用性质 = 未知                                   │
│        │                                            │
│        ↓                                            │
│   直接拒绝，提示"未配置的费用性质"                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

---

## 四、"在预算内"的判定逻辑

成本中心、费用档案、项目，判定方式相同：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   用这个ID去内存缓存里查                            │
│        │                                            │
│   ┌────┴────┐                                       │
│   找到    没找到                                    │
│   │         │                                       │
│   ↓         ↓                                       │
│  通过    调合思API查上级节点                        │
│              │                                      │
│         ┌────┴────┐                                 │
│         找到    没找到                              │
│         │         │                                 │
│         ↓         ↓                                 │
│        通过    继续查上级的上级...                  │
│                    │                                │
│               ┌────┴────┐                           │
│               找到    追溯5层都没找到               │
│               │         │                           │
│               ↓         ↓                           │
│              通过    不在预算内                      │
│                                                     │
└─────────────────────────────────────────────────────┘
```

> **项目特例**：如果缓存里没找到，但项目名包含"公摊成本"四个字，也视为通过。

## 五、校验结果汇总

| 费用性质 | 校验项 | 通过条件 | 拒绝原因 |
|---------|--------|---------|---------|
| **业务** | 成本中心 + 费用档案 | 两项都在预算内 | 缺哪项提示哪项 |
| **管理** | 成本中心 + 费用档案 | 两项都在预算内 | 缺哪项提示哪项 |
| **生产（普通）** | 项目 + 成本中心 + 费用档案 | 三项都在预算内 | 缺哪项提示哪项 |
| **生产（公摊）** | 成本中心 + 费用档案 | 两项都在预算内 | 缺哪项提示哪项 |
| **未知** | 无 | 不可能通过 | 未配置的费用性质 |

## 六、返回给员工的拒绝消息示例

| 场景 | 拒绝消息 |
|------|---------|
| 成本中心不在预算内 | "成本中心 北京研发中心(ID01xxx) 不在成本中心预算包内" |
| 费用档案不在预算内 | "明细1 差旅费住宿(ID02xxx) 不在成本中心预算包内" |
| 项目不在预算内 | "项目 XX产品研发(ID03xxx) 不在项目预算包内" |
| 成本中心不在项目下 | "成本中心 北京研发中心(ID01xxx) 不在项目预算包内" |
| 生产费用缺项目 | "生产费用缺少项目" |
| 费用性质未知 | "未配置的性质ID: ID04xxx" |

## 七、需要确认的问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | 费用性质目前只有业务/管理/生产三种，后续会加新的吗？ | 加新性质需要更新代码映射表 |
| 2 | 业务和管理费用不看项目，确认是业务规则？ | 如果需要看项目，逻辑要改 |
| 3 | "公摊成本"靠名字匹配，有没有可能叫别的名字？ | 可以配置豁免项目ID列表，不依赖名字 |
| 4 | 成本中心/费用档案找不到时向上追溯5层，够不够？ | 层级更深会校验失败 |
| 5 | 生产费用没有项目字段时（为空），当前逻辑是拒绝 | 需确认是否符合业务预期 |
| 6 | 项目下没有成本中心要求时，跳过成本中心校验 | 需确认是否符合业务预期 |

## 八、配置示例

```yaml
# 预算同步白名单（只同步这些预算包）
budget_targets:
  - id: "ID01T5nPu92vH9"    # 2026成本中心预算包
    name: "成本中心预算"
    depth: 99               # 钻取深度，99=全部层级

  - id: "ID01RFKVaUa4q3"    # 项目预算包
    name: "项目预算"
    depth: 99

# 豁免项目ID列表（这些项目走公摊豁免分支）
exempt_projects:
  - "ID01XXXXXX"    # 公摊成本项目
```
