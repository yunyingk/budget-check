# v0.8.0 预算校验逻辑重设计（硬编码版）

## 概述

原有 2 个预算包（成本中心预算、项目预算）扩展为 3 个：

1. **成本中心预算**（必须占用）
2. **板块预算**（必须占用，只校验板块预算中的成本中心）
3. **项目预算**（当费用性质=生产时占用）

---

## 一、成本中心预算（必须占用）

每次校验都必须命中成本中心预算包。

| 预算维度 | 说明 |
|---|---|
| 成本中心 | 等于或子级 |
| 费用类型 | 等于 |

### 费用类型控制方式

费用类型在成本中心预算节点下分两种控制模式：

- **单独控制**：费用类型档案A — 单独占一个子节点
- **打包控制**：打包控制1 包含 费用类型档案B、费用类型档案C — 多个费用类型归在一个子节点下，命中其中任一即可

```
成本中心预算包根节点
├── 成本中心A
│   ├── 费用类型档案A          ← 单独控制
│   ├── 打包控制1
│   │   ├── 费用类型档案B
│   │   └── 费用类型档案C
│   ├── 打包控制2
│   │   ├── 费用类型档案D
│   │   └── 费用类型档案E
│   └── 打包控制3
│       ├── 费用类型档案F
│       └── 费用类型档案G
└── 成本中心B
    └── ...
```

### 校验逻辑

1. 成本中心必须在树根的后代中（等于或子级）
2. 匹配到成本中心节点后，检查费用类型：
   - 遍历该成本中心节点的所有子节点
   - 如果子节点是叶子（单独控制）：直接比对费用类型
   - 如果子节点有子节点（打包控制）：费用类型在子节点的子节点中匹配即可
3. 费用类型为空时跳过费用类型校验

### 分摊场景

- 分摊场景通过分摊的成本中心占用预算（即使用 `apportionForm.E_system_costcenter`，而非 form 顶层）

---

## 二、板块预算（必须占用，只校验板块预算中的成本中心）

| 预算维度 | 说明 |
|---|---|
| 成本中心 | 等于或子级 |

### 校验逻辑

1. 成本中心必须在板块预算包树根的后代中
2. 只校验成本中心，不校验费用类型

### 分摊场景

- 同成本中心预算，通过分摊的成本中心占用

---

## 三、项目预算（当费用性质=生产时占用）

| 预算维度 | 说明 |
|---|---|
| 项目 | 等于 |
| 成本中心 | 等于 |

### 校验逻辑

1. 费用性质=生产时才命中此预算包
2. 项目必须在项目预算包树根的后代中（等于或子级）
3. 匹配到项目节点后，如果项目节点有子节点（成本中心），则成本中心也必须在子节点中
4. 项目不在预算包内的单据不让提交

### 分摊场景

- 项目预算无需考虑分摊场景（项目的值直接从 form 顶层取，不从 apportionForm 取）

## 四、整体校验流程（硬编码）

```
对每个 checkUnit:

  // 1. 成本中心预算（必须）
  action1 = checkCostCenterBudget(unit)

  // 2. 板块预算（必须）
  action2 = checkBoardBudget(unit)

  // 3. 项目预算（费用性质=生产时）
  if unit.expenseNature == "生产":
    action3 = checkProjectBudget(unit)
  else:
    action3 = "accept"

  三个都 accept → accept
  任一 refuse → refuse（拼接所有原因）
```

## 五、与当前代码的差异

| 项目 | 当前逻辑 | 新逻辑 |
|---|---|---|
| 预算包数量 | 2（成本中心、项目） | 3（成本中心、板块、项目） |
| 成本中心预算 | 业务/管理命中 | 所有场景必须命中 |
| 费用类型 | 不区分单独/打包 | 区分单独控制和打包控制 |
| 板块预算 | 无 | 新增，只校验成本中心 |
| 项目预算 | 生产命中 | 生产命中，项目内成本中心=等于 |
| 豁免项目 | exemptProjects 硬编码 | 项目不在预算包内直接 refuse（不在豁免列表的走公摊，仍需命中成本中心预算和板块预算） |
| 分摊 | 支持 | 成本中心/板块预算支持，项目预算不支持 |

## 八、打包控制实现方案

成本中心节点下的子节点结构：

```
成本中心A (Node)
├── 费用类型档案A (Node)          ← 叶子节点 = 单独控制
├── 打包控制1 (Node)
│   ├── 费用类型档案B (Node)
│   └── 费用类型档案C (Node)
└── 打包控制2 (Node)
    ├── 费用类型档案D (Node)
    └── 费用类型档案E (Node)
```

遍历成本中心节点的直接子节点：
- 子节点是叶子（无 Children）→ 单独控制，直接匹配费用类型
- 子节点有 Children → 打包控制，遍历其子节点匹配费用类型

现有 `FindAncestorInTree` 可复用：将打包控制节点的子节点 ID 收集成 set，调用 `FindAncestorInTree(feeType, set, 1)` 即可。
